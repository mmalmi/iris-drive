Browser footer

This page is deliberately tall so the Iris Drive WebView footer can collapse and expand under UI test.

Scrollable content

The bottom controls should feel like a compact mobile browser surface, not an app modal header.

Collapsed state

After scrolling down, only the compact site label should remain visible above the safe area.

Expanded state

After scrolling up or tapping the compact label, the back button, address field, reload button, and menu return.

End marker

Enough height to exercise real WebView scrolling on phone-sized simulators.

Glass check

The footer should blur this color through the UIKit glass, not hide it under a flat white fill.
